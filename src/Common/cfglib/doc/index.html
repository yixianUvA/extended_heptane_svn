Library overview

Cfglib is a standalone library for manipulation of control-flow-graphs (CFGs).It allows to read/write CFGs in XML files and manipulate CFGs in memory. An extensible attribute attachment mechanism allows to attach arbitrary information to any object in the library. The library manages two classes of objects:

  • First-class objects (Program, Cfg - Control Flow Graphs - , Node, Instruction, Loop). All these objects are serialisable (can be read/written in XML files)
  • Attributes: attributes can be dynamically attached to first-class objects.Attributes are classified into two categories:
    • NonSerialisable attributes: they do not provide an I/O methods, and are expected to be removed by the user
    • Serialisable attributes: they provide I/O methods to/from XML files.

The library provides a small number of built-in attributes (string, integers, unsigned long, big integers). New attributes types can be defined at run-time, provided that the name of the attribute type is not already used.

First-class objects

  • Program: represents an execution unit. A program has a name, an entry point (Cfg) and contains a set of Cfgs.
  • Cfg: represents a function. A Cfg has a name (name of calling function) and contains some Nodes (basic blocks) and Edges (possible control flow between basic blocks). The Cfg class contains most of the entry points of the library (eg: getting successors/predecessors of a Node, set of Loops, etc.). External Cfgs denote code defined elsewhere (e.g. in a library).
  • Node: represent a basic block. A Node contains a set of instructions (possibly including function call instructions). Every node has a type: it can can be a BB (basic block, if none of its instruction is a function call) or a Call otherwise.A Call node contains information on the callee (callee's name and pointer on callee's Cfg).
  • Edge: intra-cfg control-flow information (call/return relationships are described in the extra-information contained in Call nodes).
  • Loop: natural loop. Defined by a set of nodes (every node in the loop, including those embedded in subloops), entry nodes and back-edges.

All first-class objects are serialisable (derive from class Serialisable).

Attributes

Attributes can be dynamically attached to any object deriving from class Attributed (basically, every first-class object). Every attribute has a type, a name and a value. Only one attribute with a given name can be attached to an Attributed object. Arbitrarily complex attributes can be added to the library. A method of class AttributesFactory allows to add new attribute types to the library.

Compiling and installing

There is one static libray file produced, in directory build_dir. There is one GNUMakefile in the cfglib directory. The makefile compiles for Linux or MacOS hosts.

  • Compiling: type make. Produces a file libcfg.a in directory build_dir
  • Installing: modify variable INSTALL_DIR in file GNUMakefile and type make install. This will install the library libcfg.a and its documentation in directory INSTALL_DIR (sub-directories include for headers, lib for library and doc for documentation). File INSTALL_DIR/include/CfgLib.h will include all the headers required to use the library.

The only required library is libxml2. File GNUMakefile should be modified accordingly to specify the installation directory of libxml2.

Hierarchy of sources

  • include: library headers
    • Attributed.h: interface of attribued objects (all first)class objects
    • Instruction.h: interface of instructions
    • Attributes.h: common interface of attributes, that can be attached to every object deriving from class Attributed
    • BigIntegerLibrary.h: interface of BigInteger library
    • Loop.h: interface of Loop objects
    • Cfg.h: Interface of CFG objects
    • Node.h: interface of node objects
    • CfgLib.h: includes every other include file, should be included when using the library
    • NonSerialisableAttributes.h: internal (non serialisable) attributes
    • Program.h: interface of program objects
    • Edge.h: interface of edge objects
    • Factory.h: interface of factories, used to create new serialisable objects according to their types
    • Serialisable.h: common interface of all serialisable objects (all first-class objects + serialisable attributes)
    • SerialisableAttributes.h: interface of serialisable attributes
    • Handle.h: used to manage correspondance between identifiers of serialisable objects and their address in memory
    • Helper.h: useful functions
    • XmlExtra.h: small wrapper around libxml2
    • CloneHandle.h: cloning of CFGs
    • PointerAttributes.h: pointer attribute
  • src: library source code
    • One source file per header in directory include. Implements the interfaces of all objects managed by the libray
  • generated-doc: library documentation
  • build_dir: location where libcfg.a is put before installing the library
  • external_lib: source code of an open-source BigInteger library
